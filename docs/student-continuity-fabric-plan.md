# Student Continuity Fabric Plan

## Positioning

The strongest move is to stop presenting the WGU work as only a strong enterprise architecture website. Package it as a named invention with proof:

**The Student Continuity Fabric** - a governed event, identity, timeline, and action layer that prevents online students from being dropped between systems.

This is not "AI for higher ed." That phrase is too generic and too vendor-shaped. The named pain is sharper: black holes, silent changes, fragmented truth, no owner, unreliable access, and unsafe AI action.

## Category Thesis

Online institutions lose students at system seams. The Student Continuity Fabric governs those seams by combining:

- Governed student timeline
- Competency ledger
- Identity spine
- Event contracts
- Owner and SLA projections
- AI action gateway

The public promise is simple: no black holes, nothing silent, no ownerless queue, and no ungoverned AI action.

## Proof Slice

Lead with one slice before showing the full architecture:

Transcript/document intake -> student-visible status -> named owner -> SLA timer -> escalation -> audit trail.

This is the clearest proof because an executive can understand it in ten seconds, an engineer can test it, and a student can feel the difference.

## Audience Paths

President / Provost:
Mission, persistence, student trust, and economics.

CIO / CTO:
System boundary, integration pattern, build/buy posture, and scale math.

CISO / General Counsel / Registrar:
FERPA, audit, system of record clarity, and AI action tiers.

## Novelty Wedge

The invention is not a replacement for the systems an online university already needs:

- Salesforce manages relationships.
- SIS owns the academic record.
- Databricks analyzes data.
- ServiceNow and Genesys manage operational workflows.
- The Student Continuity Fabric governs the seams between them and gives students continuity.

That seam-governance layer is the wedge.

## Public Artifact

Publish the downloadable whitepaper:

**The Student Continuity Fabric: A Reference Architecture for Online Competency-Based Institutions**

It includes failure modes, target architecture, proof slice, trust model, scale math, adoption path, evidence ledger, and IP caution.

## Proof Repository Standard

The reference implementation should become a proof object with both executive and engineering evidence:

- Public README
- Architecture diagram
- Nine passing tests
- Demo script
- Screenshots or GIF
- "Black hole closed" walkthrough

Executives trust screenshots. Engineers trust tests. The package needs both.

## Evaluation Rubric

Turn the concept into a buying and architecture test:

**Can your current CRM/SIS/data stack pass the Student Continuity test?**

Score against:

- No silent changes
- Named owner on every queue
- Student-visible lifecycle
- Replayable audit
- Per-action AI authorization
- System-of-record clarity
- Cost per student per term
